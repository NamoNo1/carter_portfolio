General Market — Shop

ระบบร้านค้าในเกม (NUI) ธีมมืดสไตล์แดชบอร์ด แบ่งโซนยอดเงิน หมวดสินค้า กริดไอเทม ตะกร้า และเลือกชำระด้วยเงินสดหรือธนาคาร พร้อมปุ่มสั่งซื้อ

ได้ทำงานร่วมกับทีมออกแบบ UI จาก Figma โดยประสานงานจากไฟล์ดีไซน์มาพัฒนาเป็นหน้า NUI ให้ตรงเลย์เอาต์ สี คอมโพเนนต์ และสถานะต่าง ๆ ตามที่ทีมดีไซน์กำหนด

เทคโนโลยี: เขียนด้วย Lua (ฝั่ง FiveM client/server), Vue.js จัด state ของตะกร้าและ UI, บันเดิลด้วย Vite, สไตล์ด้วย Tailwind CSS ร่วมกับ HTML และ CSS, JavaScript เชื่อมสื่อสารกับ Lua ผ่าน NUI

วิธีการพัฒนา: เมื่อเปิดร้าน Lua ฝั่ง client ส่งข้อมูลสินค้าและยอดเงินผู้เล่นเข้า Vue ผ่านข้อความ NUI ผู้ใช้เลือกหมวด ค้นหา และเพิ่มของในตะกร้า — Vue คำนวณยอดรวมในเบราว์เซอร์โดยไม่ต้องวนกลับเซิร์ฟทุกคลิก กดซื้อแล้ว frontend ส่ง callback กลับไปที่ Lua client แล้วต่อไปยัง server เพื่อตรวจสอบเงิน หักบัญชี อัปเดตฐานข้อมูล และมอบไอเทม — แยกชั้นชัดระหว่าง UI (Vue) กับ logic เกม (Lua) ตามแพตเทิร์น NUI ของ FiveM

ได้ทำงานร่วมกับทีมออกแบบ UI จาก Figma นำสเปกเลย์เอาต์และคอมโพเนนต์จากดีไซน์มา implement เป็นหน้า NUI ให้ใช้งานจริงในเกม

เทคโนโลยี: Lua ฝั่ง FiveM (client/server) เปิด UI และซิงค์ข้อมูลจดหมาย, Vue.js จัด state รายการและรายละเอียดจดหมาย, Vite สำหรับบันเดิล frontend, Tailwind CSS ร่วมกับ HTML และ CSS สำหรับเลย์เอาต์และธีม, JavaScript เชื่อม NUI กับ Lua

วิธีการพัฒนา: เมื่อเปิดกล่องจดหมาย Lua client ดึงข้อมูลจาก server แล้วส่งรายการจดหมายเข้า Vue ผ่าน NUI message ผู้เล่นเลือกอ่าน ลบ หรือรับของแนบ — การกระทำที่มีผลกับข้อมูลในเซิร์ฟเวอร์จะส่ง callback จาก Vue ไปยัง Lua แล้วให้ server ตรวจสอบสิทธิ์ อัปเดตฐานข้อมูล และตอบกลับผลให้ UI รีเฟรชสถานะ แยกการแสดงผล (Vue) กับ business logic และความปลอดภัย (Lua/server) ชัดเจน

วิธีการพัฒนา: ตอนเปิด UI server ส่งสถานะผู้เล่นใน Battle Pass ปัจจุบัน (เลเวล, EXP, รางวัลที่กดรับแล้ว, ซื้อพรีเมียมหรือยัง) เข้า Vue เพื่อเรนเดอร์แทร็กและปุ่มต่าง ๆ การกดรับรางวัลหรือปลดล็อกพาสจะส่ง event จาก Vue ไป Lua client แล้วไป server เพื่อตรวจเงื่อนไข อัปเดตฐานข้อมูลและแจกของ — ข้อมูลสำคัญไม่เชื่อถือจากฝั่ง UI อย่างเดียว แยกการแสดงผลกับ logic รางวัลชัดเจน
